# DevLite 功能说明

## 核心使用场景

DevLite 的核心场景是服务本地项目迭代开发：用户在 `localhost`、`127.0.0.1` 或团队本地预览服务启动的页面中，直接选择元素、临时调整样式/文案/图片，然后复制修复 Prompt 交给本地项目里的 Agent 落到源码。

DevLite 不把页面上的 selector 当成最终实现方案。导出的 Prompt 会把 selector、DOM 路径、文本片段、命中的 CSS 规则和资源线索提供给 Agent，让 Agent 回到当前本地项目中查找真实组件、样式文件、文案和图片资源。

## 诊断模式

用户点击「开始诊断」后，DevLite 会对当前页面开始采集诊断事件。

支持事件：

- JS 运行错误
- 未处理 Promise 异常
- `console.error`
- `fetch` 请求状态
- `XMLHttpRequest` 请求状态
- `WebSocket` / `EventSource` 连接状态
- 请求失败
- 慢请求
- 静态资源加载失败
- 用户点击路径
- 基础性能快照
- 主线程长任务
- 大资源和慢资源

点击「停止诊断」后，DevLite 不再采集新事件。

## 中英文切换

DevLite 默认使用中文。页面诊断栏底部的语言按钮位于设置图标旁边，可在中文和英文之间切换。

切换后会影响：

- options 页面文案
- 页面右侧常驻图标和诊断栏文案
- 元素编辑浮层文案
- Markdown 报告
- 本地规则分析结果
- 修复 Prompt 的任务说明
- 性能诊断 Prompt 的任务说明和问题描述

配置会保存在浏览器本地，后续页面会沿用当前语言。

## 数据获取

「数据获取」页会以简洁列表展示最近请求，包括 `fetch`、`XMLHttpRequest`，以及真实业务 `WebSocket` / `EventSource` 的连接状态。点击单条请求后，可以查看：

- Preview：优先把 JSON response 解析成更直观的层级预览
- Response：查看原始响应内容
- Request：查看请求方法、URL 和 request body
- Headers：查看可获取到的 request headers、response headers 和元数据

打开面板后会对后续请求开启有限长度的 response body 摘要采集；历史请求如果没有采到内容，需要重新触发目标请求。Vite HMR 等本地开发工具长连接默认隐藏，避免占据请求列表；需要排查开发连接时，可在数据获取页打开「开发流量」查看连接生命周期事件。开发长连接的 message 帧不会入库。

## 性能诊断

「性能诊断」页用于快速判断页面是否存在卡顿和加载异常，包含：

- DOMContentLoaded、Load、资源体积和长任务数量
- 首包时间偏高
- 页面加载阶段偏慢
- 主线程长任务
- 大资源加载
- 慢资源或慢请求
- 静态资源加载失败

点击「复制性能 Prompt」可以把页面指标、问题列表、证据和建议整理成结构化 Prompt，直接交给 agent 定位和修复。

## 报告生成

点击「生成报告」后，DevLite 会生成 Markdown 报告，包含：

- 问题摘要
- 页面信息
- 本地规则分析
- 事件统计
- 错误和请求明细
- 页面修改
- 隐私说明

报告可以直接复制给开发者、客服或 agent 工具。

## 元素选择器

点击页面右侧 DevLite 图标或浏览器工具栏中的 DevLite 图标后，页面会出现 DevLite 浮层。

交互方式：

1. 点击页面右侧 DevLite 图标，打开始终置顶的诊断栏。
2. 拖动诊断栏顶部，可把它移动到页面任意合适位置。
3. 鼠标移动到元素上，显示高亮边框。
4. 点击目标元素，打开页面编辑浮层。
5. 修改 CSS 或文字内容后，页面实时变化。
6. 点击「复制修复 Prompt」可直接复制实现任务。

支持修改：

- HTML 文字内容
- 文字颜色
- 背景颜色
- 字号
- 字重
- 行高
- 字距
- padding
- margin
- width
- height
- border-radius
- box-shadow
- display
- gap
- justify-content
- align-items
- opacity

所有修改都只作用于当前页面，刷新页面后恢复。CSS 会以临时 inline style 形式生效，文字内容会直接改写当前 DOM 文本。

## 图片替换与裁剪

点击「替换图片」后，DevLite 会读取用户选择的图片。对于 PNG、JPG、WebP 等 raster 图片，会先打开裁剪工具；用户可以按当前元素比例裁剪，也可以切换自由比例、缩放和重置。SVG 文件继续直接替换，不进入 canvas 裁剪。

复制修复 Prompt 时，DevLite 不会把裁剪后的 base64 图片正文塞进 Prompt。Prompt 会记录：

- 页面原本的图片资源 URL 或项目路径。
- 用户上传文件的文件名、类型、大小和原始尺寸。
- 裁剪框位置、裁剪尺寸、输出尺寸和输出类型，用于表达视觉裁剪意图。
- 给 Agent 的本地项目资源查找指令。

Agent 应优先在当前本地项目中查找原页面资源路径或同名上传文件。如果项目里已有资源，以裁剪信息作为视觉参考，生成或引用项目内图片并替换源码中的 `src`、`srcset`、SVG `href` 或背景图路径；如果项目里没有该文件，应提示用户先把图片放入项目资源目录，而不是凭空生成无关图片。

## 描述需求

元素编辑浮层里的「描述需求」用于记录无法直接通过临时 CSS、文案、图片或 DOM 操作表达的修改意图。选择元素后会默认显示输入框，用户也可以通过「描述需求」按钮重新展开，描述希望 Agent 实现的设计或功能调整。

- 「立即复制」会立刻复制当前元素的结构化 Prompt，并把已记录的修改放入「待验证」。
- 「稍后复制」会打开元素面板；如果当前元素已有修改或描述内容，会进入「待处理」，之后点击「复制修复 Prompt」时和其他元素修改一起导出。
- 描述内容可以为空。为空时，导出的 Prompt 只包含该元素已经直接完成的样式、文案、图片或结构修改。
- 导出的 Prompt 会把需求放在对应元素的 `modifications` 中，类型为 `request`，并附带 selector、DOM 路径、文本摘要和命中的 CSS 规则作为源码定位线索。
- 含描述需求的记录不会自动归档。即使页面热更新后其他可验证修改已经匹配，也需要用户在「待验证」中手动点击「标记已修复」确认归档。

## 导出格式

### 修复 Prompt

面向本地代码实现和问题分析，强调：

- 当前页面通常来自本地开发服务器，例如 `localhost` 或 `127.0.0.1`
- 先阅读项目结构
- 不假设文件路径
- 定位真实组件和样式文件
- 定位真实图片资源和项目 assets 路径
- 按项目现有技术栈实现
- 保持响应式
- 运行已有检查命令
- 分析错误原因
- 将 CSS 修改和文字内容修改转换为源码实现建议
- 将描述需求转换为源码中的设计或功能调整
- 不把 selector 当成最终实现方案
- 根据项目技术栈转换样式写法

### Markdown

完整诊断报告，适合发给开发者或保存为 issue。

### JSON

结构化会话数据，适合后续自动化处理。
