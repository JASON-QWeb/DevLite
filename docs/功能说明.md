# DevLite 功能说明

## 诊断模式

用户点击「开始诊断」后，DevLite 会对当前页面开始采集诊断事件。

支持事件：

- JS 运行错误
- 未处理 Promise 异常
- `console.error`
- `fetch` 请求状态
- `XMLHttpRequest` 请求状态
- 请求失败
- 慢请求
- 静态资源加载失败
- 用户点击路径
- 基础性能快照
- 主线程长任务
- 大资源和慢资源

点击「停止诊断」后，DevLite 不再采集新事件。

## 中英文切换

DevLite 默认使用中文。页面诊断栏底部的语言按钮位于设置图标旁边，可在中文和英文之间切换。

切换后会影响：

- popup 和 options 页面文案
- 页面右侧常驻图标和诊断栏文案
- 元素编辑浮层文案
- Markdown 报告
- 本地规则分析结果
- AI Prompt 的任务说明
- 性能诊断 Prompt 的任务说明和问题描述

配置会保存在浏览器本地，后续页面会沿用当前语言。

## 数据获取

「数据获取」页会以简洁列表展示最近请求。点击单条请求后，可以查看：

- Preview：优先把 JSON response 解析成更直观的层级预览
- Response：查看原始响应内容
- Request：查看请求方法、URL 和 request body
- Headers：查看可获取到的 request headers、response headers 和元数据

默认不采集 response body。需要查看 Preview 或 Response 时，可在请求详情里开启响应体采集，然后重新触发目标请求。

## 性能诊断

「性能诊断」页用于快速判断页面是否存在卡顿和加载异常，包含：

- DOMContentLoaded、Load、资源体积和长任务数量
- 首包时间偏高
- 页面加载阶段偏慢
- 主线程长任务
- 大资源加载
- 慢资源或慢请求
- 静态资源加载失败

点击「复制性能 Prompt」可以把页面指标、问题列表、证据和建议整理成结构化 Prompt，直接交给 agent 定位和修复。

## 报告生成

点击「生成报告」后，DevLite 会生成 Markdown 报告，包含：

- 问题摘要
- 页面信息
- 本地规则分析
- 事件统计
- 错误和请求明细
- 页面修改
- 隐私说明

报告可以直接复制给开发者、客服或 AI 工具。

## 元素选择器

点击页面右侧 DevLite 图标或 popup 中的「选择元素」后，页面会出现 DevLite 浮层。

交互方式：

1. 点击页面右侧 DevLite 图标，打开始终置顶的诊断栏。
2. 拖动诊断栏顶部，可把它移动到页面任意合适位置。
3. 鼠标移动到元素上，显示高亮边框。
4. 点击目标元素，打开页面编辑浮层。
5. 修改 CSS 或文字内容后，页面实时变化。
6. 点击「复制 AI Prompt」可直接复制实现任务。

支持修改：

- HTML 文字内容
- 文字颜色
- 背景颜色
- 字号
- 字重
- 行高
- 字距
- padding
- margin
- width
- height
- border-radius
- box-shadow
- display
- gap
- justify-content
- align-items
- opacity

所有修改都只作用于当前页面，刷新页面后恢复。CSS 会以临时 inline style 形式生效，文字内容会直接改写当前 DOM 文本。

## 导出格式

### AI Prompt

面向本地代码实现和问题分析，强调：

- 先阅读项目结构
- 不假设文件路径
- 定位真实组件和样式文件
- 按项目现有技术栈实现
- 保持响应式
- 运行已有检查命令
- 分析错误原因
- 将 CSS 修改和文字内容修改转换为源码实现建议
- 不把 selector 当成最终实现方案
- 根据项目技术栈转换样式写法

### Markdown

完整诊断报告，适合发给开发者或保存为 issue。

### JSON

结构化会话数据，适合后续自动化处理。

## 回归验证

项目内置一个 demo 页面用于真实浏览器回归测试：

- 目录：`demo/devlite-qa`
- 命令：`npm run qa:extension`

该命令会先构建扩展，再启动 demo 服务和临时 Chrome profile，加载本地 `dist` 扩展并自动验证：

- 右侧常驻图标是否在本地页面显示
- 点击图标后是否开始采集并打开诊断栏
- 中英文切换是否生效
- 设置图标是否打开 options 页面
- fetch、XHR、失败请求、慢请求和 response preview 是否可见
- Preview、Response、Request、Headers 详情是否可切换
- JS 错误和 Promise 异常是否进入诊断面板
- 性能诊断和性能 Prompt 是否可复制
- 元素样式修改和双击文字编辑是否生效
- 全量修改 Prompt 是否可复制
- 后台 Markdown 报告是否可生成

如果系统 Chrome 禁止命令行加载 unpacked extension，可先安装 Chrome for Testing：

```bash
npx @puppeteer/browsers install chrome@stable --path /tmp/devlite-browsers
```

脚本会优先使用 `/tmp/devlite-browsers` 中的 Chrome for Testing，也可以通过 `CHROME_BIN=/path/to/chrome npm run qa:extension` 指定浏览器。
