# DevLite 功能说明

## 诊断模式

用户点击「开始诊断」后，DevLite 会对当前页面开始采集诊断事件。

支持事件：

- JS 运行错误
- 未处理 Promise 异常
- `console.error`
- `fetch` 请求状态
- `XMLHttpRequest` 请求状态
- 请求失败
- 慢请求
- 静态资源加载失败
- 用户点击路径
- 基础性能快照

点击「停止诊断」后，DevLite 不再采集新事件。

## 报告生成

点击「生成报告」后，DevLite 会生成 Markdown 报告，包含：

- 问题摘要
- 页面信息
- 本地规则分析
- 事件统计
- 错误和请求明细
- 页面修改
- 隐私说明

报告可以直接复制给开发者、客服或 AI 工具。

## 元素选择器

点击页面右侧 DevLite 图标或 popup 中的「选择元素」后，页面会出现 DevLite 浮层。

交互方式：

1. 点击页面右侧 DevLite 图标，打开始终置顶的诊断栏。
2. 拖动诊断栏顶部，可把它移动到页面任意合适位置。
3. 鼠标移动到元素上，显示高亮边框。
4. 点击目标元素，打开页面编辑浮层。
5. 修改 CSS 或文字内容后，页面实时变化。
6. 点击「复制 AI Prompt」可直接复制实现任务。

支持修改：

- HTML 文字内容
- 文字颜色
- 背景颜色
- 字号
- 字重
- 行高
- 字距
- padding
- margin
- width
- height
- border-radius
- box-shadow
- display
- gap
- justify-content
- align-items
- opacity

所有修改都只作用于当前页面，刷新页面后恢复。CSS 会以临时 inline style 形式生效，文字内容会直接改写当前 DOM 文本。

## 导出格式

### AI Prompt

面向本地代码实现和问题分析，强调：

- 先阅读项目结构
- 不假设文件路径
- 定位真实组件和样式文件
- 按项目现有技术栈实现
- 保持响应式
- 运行已有检查命令
- 分析错误原因
- 将 CSS 修改和文字内容修改转换为源码实现建议
- 不把 selector 当成最终实现方案
- 根据项目技术栈转换样式写法

### Markdown

完整诊断报告，适合发给开发者或保存为 issue。

### JSON

结构化会话数据，适合后续自动化处理。
