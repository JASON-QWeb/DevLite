# DevLite 架构设计

## 产品定位

DevLite 不是开发者专用的 F12 替代品，而是面向本地项目迭代开发的网页问题诊断和可视化改样工具。它的核心使用方式是：用户在 `localhost`、`127.0.0.1` 或团队本地预览服务打开当前项目页面，在浏览器里直接调整元素，再把结构化 Prompt 交给本地项目 Agent 落到源码。

核心目标：

1. 普通用户不用打开开发者工具，也能生成可交给开发者或本地 Agent 的错误报告。
2. 前端开发者可以直接在本地开发服务器页面上调整 CSS、文案和图片，并导出为可实现的任务说明。
3. 默认本地分析，减少隐私风险和上架审核压力。

## 模块职责

### 页面入口

普通 HTTP/HTTPS 页面会通过 manifest 静态注入 `content.js`，默认显示右侧 DevLite 入口。用户可以从页面入口打开诊断面板或启动元素选择器。

页面入口负责：

- 打开和关闭页面诊断面板
- 启动元素选择器
- 展示错误、请求、性能和样式修改信息
- 生成并复制修复 Prompt
- 临时应用页面样式和文字修改

### toolbar action

浏览器工具栏图标不再打开 popup 页面。点击图标时，background 直接在当前 tab 打开页面内 DevLite 面板，负责：

- 打开当前页面面板
- 确保 content script 和 injected script 已注入
- 当前页面尚无 session 时创建诊断会话
- 不额外打开扩展页面，避免用户看到独立 popup UI
- 复制修复 Prompt / Markdown / JSON 导出

### options

options 负责长期配置：

- 语言
- 是否采集 response 摘要
- response 摘要最大长度
- 慢请求阈值
- 额外脱敏字段

### background service worker

background 是数据和能力中心：

- 按 tab 管理诊断会话
- 注入或动态注册 MAIN world 的 injected script
- 聚合诊断事件
- 聚合样式修改记录
- 执行脱敏
- 执行本地规则分析
- 生成报告和 Prompt

### content script

content script 运行在 Chrome 扩展隔离环境中：

- 接收 injected script 通过 `window.postMessage` 发来的事件
- 捕获 DOM 资源加载失败
- 记录用户点击
- 渲染页面右侧诊断栏入口
- 渲染元素选择高亮层
- 渲染可拖动 CSS 编辑浮层
- 应用临时 inline style
- 记录 before / after 样式修改

content script 不能直接访问页面 JavaScript 运行环境，因此不负责 hook 页面原生的 `fetch`、`XMLHttpRequest`、`WebSocket` 和 `EventSource`。它会给与 injected script 的 `window.postMessage` 消息附加当前页面会话 token，并只接收匹配 token 的 DevLite 事件。Vite HMR 等开发工具长连接会被标记为开发流量：连接生命周期可按需查看，message 帧默认丢弃，避免刷屏和占用会话容量。

### injected script

injected script 通过 `chrome.scripting.registerContentScripts` 或 `chrome.scripting.executeScript` 注入到页面 `MAIN` world：

- hook `window.fetch`
- hook `XMLHttpRequest`
- hook `WebSocket`
- hook `EventSource`
- hook `console.error`
- 监听 `window.error`
- 监听 `unhandledrejection`

采集到的数据通过带 token 的 `window.postMessage` 发给 content script，再由 content script 发给 background。

## 国际化架构

DevLite 支持中文和英文切换，语言配置保存在 `chrome.storage.local` 的 settings 中，默认中文。

文案按运行环境分层管理：

- `public/_locales/`：Chrome manifest 的扩展名称和描述，默认 `zh_CN`，并提供英文描述。
- `src/shared/i18n.ts`：options、报告入口等扩展页面共享 UI 文案，提供 `createTranslator(locale)` 和 `uiText(locale, key)`。
- `src/content/i18n.ts`：页面右侧常驻图标、页面诊断栏、元素编辑浮层等 content script UI 文案。
- `src/shared/analyzer.ts`：本地规则分析结果，按 locale 输出摘要、问题、证据和建议。
- `src/shared/report.ts`：Markdown 报告结构和字段名，按 locale 输出中英文报告。
- `src/shared/exporters.ts`：修复 Prompt 的任务说明，按 locale 输出，但字段结构保持稳定，便于 agent 解析。

content script 不直接复用 `src/shared/i18n.ts` 的运行时代码。原因是 Manifest V3 的静态 content script 需要构建成无顶层 `import/export` 的独立脚本；如果把 options 共享模块直接引入 content script，Rollup 可能抽出共享 chunk，导致 content script 产物不符合当前 smoke 约束。因此 content UI 使用独立的 `src/content/i18n.ts`，但仍遵循相同规范：稳定 key、默认中文、英文 fallback 不散落在业务模板中。

新增文案时遵循：

1. options/background/report 使用 `src/shared/i18n.ts`。
2. 页面浮层 UI 使用 `src/content/i18n.ts`。
3. 诊断规则输出放在对应规则 formatter 中，并显式接收 locale。
4. 不在模板中写“中文字符串到英文字符串”的临时映射。
5. 修复 Prompt 和 JSON 导出保持结构字段稳定，只翻译任务说明和用户可读文本。

## 数据流

```text
用户点击页面入口或浏览器工具栏图标开始诊断
        ↓
content.js 静态入口已加载，background 确保 injected.js 已进入 MAIN world
        ↓
content script 启动诊断并通知 injected script
        ↓
injected script 采集 JS / Network / console 事件
        ↓
content script 采集资源失败、点击路径、CSS 修改
        ↓
background 聚合、脱敏、分析
        ↓
页面面板展示统计、报告、导出内容
```

## 样式修改记录

每次选择元素后，DevLite 会记录：

- selector
- DOM 路径
- 元素标签和 class
- 元素文本摘要
- 修改前 computed style
- 修改后 CSS 属性
- 用户输入的描述需求
- 当前视口尺寸

这些记录不会假设源码文件路径。导出修复 Prompt 时，会明确要求 agent 根据 selector、DOM 路径、文本和描述需求在当前本地项目中定位真实组件。

描述需求用于记录无法由临时页面改动直接表达的设计或功能调整。它会作为对应元素的结构化 `request` modification 导出；含描述需求的记录不会自动归档，必须由用户在待验证列表中手动确认。

## 图片资源定位

图片替换记录会保留页面原始资源线索和用户上传文件线索：

- 原页面 `src`、`srcset`、SVG `href` 或 CSS `background-image` 中的 URL/路径。
- 上传文件名、MIME、大小、原始尺寸。
- 裁剪框位置、输出尺寸和输出类型，用于表达视觉裁剪意图。
- selector、DOM 路径、父级链路和命中的 CSS 规则。

DevLite 不把裁剪后的图片 base64 正文塞进修复 Prompt。Prompt 的目标是让本地项目 Agent 回到源码中查找资源：优先搜索原页面资源路径或同名上传文件；若资源存在，则按项目约定生成或引用 assets；若资源不存在，则提示用户把图片加入项目资源目录。这样可以避免 Prompt 过大，也避免把用户本地图片内容默认复制给外部工具。

## 本地规则分析

当前内置规则包括：

- 401：登录态、Token 或鉴权问题
- 403：权限或 CSRF 问题
- 404：接口路由、资源路径或发布路径问题
- 5xx：后端服务异常
- CORS：跨域配置问题
- Mixed Content：HTTPS 页面加载 HTTP 资源
- ChunkLoadError：前端资源版本或缓存问题
- Cannot read properties：空值保护不足或接口结构变化

这些规则可离线运行。
