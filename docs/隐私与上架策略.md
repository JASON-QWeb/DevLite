# 隐私与上架策略

## 默认策略

DevLite 的默认策略是：

- 核心用途是服务本地开发服务器页面，例如 `localhost`、`127.0.0.1` 或团队本地预览服务，用于前端页面编辑、诊断和迭代开发。
- 普通 HTTP/HTTPS 页面默认显示右侧入口。
- 用户主动点击后才开始诊断。
- 只诊断当前标签页。
- 默认本地分析。
- 默认不上传数据。
- 默认不采集完整 response body。
- 默认不读取完整页面 HTML。
- 默认不持久修改页面。

这个策略是为了降低用户隐私风险，也降低 Chrome Web Store 审核风险。

## 权限解释

### content_scripts

用于在普通 HTTP/HTTPS 页面加载轻量入口脚本。该脚本默认只显示右侧 DevLite 图标，不启动接口、错误和控制台采集。

### host_permissions

保留 `http://*/*` 和 `https://*/*`，用于让 DevLite 在普通 HTTP/HTTPS 页面显示右侧入口，并在用户主动打开面板、开始诊断或选择元素后访问当前页面上下文。

这个项目不把 host 权限改成 `optional_host_permissions`。原因是 DevLite 的核心体验依赖页面常驻入口和 MAIN world 采集脚本，完全改成 `activeTab` 会让入口和预注册脚本失效，反而改变产品行为。上架时应在权限说明中明确：host 权限用于显示入口和按用户操作诊断当前页面，不用于后台自动上传或跨站读取数据。

### scripting

用于注入诊断脚本、元素选择器和页面面板。`content.js` 通过 manifest 静态注入，background 只动态注册/注入 MAIN world 的 `injected.js`。

### storage

用于保存本地设置，例如：

- 是否采集 response 摘要
- response 最大长度
- 慢请求阈值
- 额外脱敏字段

`chrome.storage.session` 的当前限制由 Chrome API 决定，诊断会话写入失败时会压缩旧会话、事件和样式记录后重试。

### 剪贴板

不声明 `clipboardWrite` 权限。扩展页面使用用户点击触发的 Clipboard API；页面浮层复制在失败时使用 `document.execCommand("copy")` 兜底。

## 数据采集范围

DevLite 可能采集：

- 当前页面 URL
- 页面标题
- 浏览器 UA
- 视口尺寸
- JS 错误信息和堆栈
- Promise 异常
- `console.error`
- 请求 URL、方法、状态码、耗时
- 资源加载失败 URL
- 最近点击元素摘要
- 用户主动修改的 CSS 属性

默认不采集：

- Cookie
- Authorization
- Set-Cookie
- 完整页面 HTML
- 密码输入框内容
- 完整 response body

## 脱敏规则

内置脱敏字段包括：

- password
- passwd
- token
- access_token
- refresh_token
- secret
- apiKey
- authorization
- cookie
- set-cookie
- csrf-token
- phone
- email
- idCard
- creditCard

同时会对邮箱、手机号、Bearer Token、常见 token 字段和银行卡号模式做文本级替换。

## Chrome Web Store 文案建议

建议描述：

> DevLite 面向本地项目迭代开发，适合在 localhost、127.0.0.1 或团队本地预览服务打开的页面上进行前端诊断和可视化修改。DevLite 会在普通 HTTP/HTTPS 页面显示右侧入口。只有用户主动打开面板、开始诊断或选择元素后，才会对当前页面临时收集前端错误、资源加载失败、接口状态和用户主动选择的元素样式/文案/图片信息，用于生成本地诊断报告和修复 Prompt。数据默认保存在本地，不会自动上传。

图片替换说明建议：

> 当用户主动选择图片并进行裁剪替换时，DevLite 会在本地浏览器中读取该图片并生成页面临时预览。导出的修复 Prompt 默认不包含图片 base64 正文，只包含原页面图片资源 URL/路径、上传文件名、图片尺寸和裁剪信息，用于帮助本地项目 Agent 在项目 assets 中定位或替换资源。

不建议描述：

> 自动监控所有网页。
> 读取所有网页数据。
> 自动上传接口响应。
> 自动修改网站源码。
> 自动读取用户本地文件路径。
> 自动上传用户选择的图片。

## 上架前检查清单

- 商店页面说明数据用途。
- 隐私政策中说明默认本地分析。
- 权限说明和实际功能一致。
- 说明 HTTP/HTTPS 页面入口常驻和诊断采集按需启动的边界。
- response body 默认关闭。
- 提供用户删除设置的方式。
