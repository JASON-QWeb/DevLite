# 隐私与上架策略

## 默认策略

DevLite 的默认策略是：

- 普通 HTTP/HTTPS 页面默认显示右侧入口。
- 用户主动点击后才开始诊断。
- 只诊断当前标签页。
- 默认本地分析。
- 默认不上传数据。
- 默认不采集完整 response body。
- 默认不读取完整页面 HTML。
- 默认不持久修改页面。

这个策略是为了降低用户隐私风险，也降低 Chrome Web Store 审核风险。

## 权限解释

### activeTab

用于在用户主动点击扩展后，临时访问当前页面。页面关闭或跳转后，该访问能力会失效。

### content_scripts

用于在普通 HTTP/HTTPS 页面加载轻量入口脚本。该脚本默认只显示右侧 DevLite 图标，不启动接口、错误和控制台采集。

### host_permissions

用于在用户点击页面入口后，按需向当前 HTTP/HTTPS 页面注入采集脚本。`file://` 页面不在默认范围内，如需支持需要用户在扩展详情中单独允许访问文件网址。

### scripting

用于注入诊断脚本和元素选择器。DevLite 需要它来捕获当前页面的运行错误、接口状态和 CSS 信息。

### storage

用于保存本地设置，例如：

- 是否采集 response 摘要
- response 最大长度
- 慢请求阈值
- 额外脱敏字段
- 用户自己的 AI API Key

### clipboardWrite

用于一键复制诊断报告、CSS 修改记录和 AI Prompt。

### optional_host_permissions

用于可选能力：

- AI API 调用只在用户启用并确认发送后请求对应服务商接口权限。

## 数据采集范围

DevLite 可能采集：

- 当前页面 URL
- 页面标题
- 浏览器 UA
- 视口尺寸
- JS 错误信息和堆栈
- Promise 异常
- `console.error`
- 请求 URL、方法、状态码、耗时
- 资源加载失败 URL
- 最近点击元素摘要
- 用户主动修改的 CSS 属性

默认不采集：

- Cookie
- Authorization
- Set-Cookie
- 完整页面 HTML
- 密码输入框内容
- 完整 response body

## 脱敏规则

内置脱敏字段包括：

- password
- passwd
- token
- access_token
- refresh_token
- secret
- apiKey
- authorization
- cookie
- set-cookie
- csrf-token
- phone
- email
- idCard
- creditCard

同时会对邮箱、手机号、Bearer Token、常见 token 字段和银行卡号模式做文本级替换。

## AI 模式

AI 默认关闭。

用户启用 AI 时，需要满足：

1. 在设置页选择「用户 API Key」模式。
2. 填写自己的 API Key。
3. 在 popup 中生成发送预览。
4. 用户点击确认后才发送。

发送内容来自已脱敏的报告和样式修改记录。

## Chrome Web Store 文案建议

建议描述：

> DevLite 会在普通 HTTP/HTTPS 页面显示右侧入口。只有用户主动打开面板、开始诊断或选择元素后，才会对当前页面临时收集前端错误、资源加载失败、接口状态和用户主动选择的元素样式信息，用于生成本地诊断报告。数据默认保存在本地，不会自动上传。AI 分析功能默认关闭，仅在用户配置自己的 API Key 并确认发送后启用。

不建议描述：

> 自动监控所有网页。
> 读取所有网页数据。
> 自动上传接口响应。
> 自动修改网站源码。

## 上架前检查清单

- 商店页面说明数据用途。
- 隐私政策中说明默认本地分析。
- 隐私政策中说明 AI 发送前确认。
- 权限说明和实际功能一致。
- 说明 HTTP/HTTPS 页面入口常驻和诊断采集按需启动的边界。
- response body 默认关闭。
- API Key 只保存在本地。
- 提供用户删除设置的方式。
